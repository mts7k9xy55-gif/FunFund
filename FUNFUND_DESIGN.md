```
┌─────────────────────────────────────────────────────────────┐
│ [FunFund]                    [Community: デザイン思考研究会] │
├────────┬────────────────────────────────────────────┬────────┤
│        │                                            │        │
│  左    │           中央：議論フィード                 │  右    │
│ サイド │                                            │ ペイン │

│        │  │ PROPOSAL Card            │              │ 評定   │
│ • 議論 │  │ 「UIデザイン原則の策定」    │              │ パネル │

│        │  └──────────────────────────┘              │ 選択中 │
```

## 2. 状態遷移フロー

### 議論 → 議題化 → 判断 → プロジェクト化

```
COMMENT（思考の流れ）
  ↓ [議題にする]ボタン
PROPOSAL（議題・テーマ）
  ↓ 議論が深まる → COMMENTが集まる
EVALUATION（判断・評定）
  ↓ 右ペインで実施 → 暗転演出
  ↓ 中央に1行記録だけ残る
PROJECT（実行フェーズ）
  ↓ 議題が具体化 → プロジェクト化
```

### フロー詳細

1. **コメント段階（COMMENT）**
   - 軽い、連続的
   - 思考の流れを記録
   - いつでも誰でも追加可能
   - 各コメントに「議題にする」ボタン

2. **議題化（PROPOSAL）**
   - コメントから昇格
   - カード形式で視覚的に強調
   - 議論の焦点になる
   - 「議論する」「評定する」のアクション

3. **判断（EVALUATION）**
   - 右ペインで実施（または全画面モーダル）
   - 3段階の儀式的演出
   - 中央フィードには「○○が判断を表明」の1行のみ
   - 不可逆・重い行為として扱う

4. **プロジェクト化**
   - 議題から生成
   - 左サイドバーのProjects欄に追加
   - 実行フェーズへ移行

## 3. COMMENT / PROPOSAL / EVALUATION の視覚設計

### COMMENT（軽）
- 最小限の装飾
- 左インデント、細いボーダー
- 流れるように読める
- ホバーで「議題にする」ボタン表示

### PROPOSAL（中）
- カード形式、白背景
- わずかな影
- タイトル＋本文
- 明確なアクションボタン
- 議論の焦点として機能

### EVALUATION（重）
- **実施中**：全画面暗転、儀式的UI
- **記録**：中央に1行テキストのみ
  - 例：「Carol が判断を表明しました」
  - クリックで詳細モーダル（オプション）
- 右ペインに評定パネル表示（選択中の議題に対して）

## 4. なぜこのUIが議論の質を高めるのか

### 1. 思考の余白を作る
- Enter=改行により、送信前に推敲できる
- 2-3行の入力欄で文章として考えられる
- 即時性より熟考を促す

### 2. 議論の焦点化
- コメントから議題への昇格フロー
- 議題がカードとして視覚的に強調される
- 「何について話しているか」が明確

### 3. 判断の重み付け
- 評定は別空間（右ペイン/モーダル）で実施
- 儀式的演出で不可逆性を表現
- 軽口と本気の判断を明確に分離

### 4. 視覚的ノイズの削減
- 色を最小限に
- 余白とタイポグラフィで階層表現
- 重さの異なる行為を同じ平面に置かない

### 5. 行動の明確化
- 「議題にする」「判断する」など動詞ベースUI
- クリック可能な要素は必ず結果を生む
- 曖昧な装飾ボタンの排除

## 5. UI用語（日本語/英語）

| 日本語 | 英語 | 説明 |
|--------|------|------|
| コメントする | Comment | 思考の流れを記録 |
| 議題にする | Propose | コメントを議題に昇格 |
| 判断する/評定する | Evaluate | 署名された判断 |
| 議論する | Discuss | 議題について話す |
| プロジェクト化 | Create Project | 議題を実行フェーズへ |

## 6. 右ペイン：評定パネル

### 表示条件
- 議題（PROPOSAL）が選択されている時のみ表示
- それ以外は非表示または閉じた状態

### 内容
- 選択中の議題タイトル
- 評定フォーム：
  - Vote: ✓ Positive / ○ Neutral / ✗ Negative
  - Stake: 数値入力
  - Reasoning: テキストエリア
- [判断を確定する] ボタン
  - クリックで儀式的演出開始

### 演出
1. ボタンクリック
2. 右ペイン消失
3. 全画面暗転（3段階）
4. 中央フィードに1行記録追加
5. 右ペイン復帰（または閉じる）

## 7. 絵文字の扱い

### 原則
- 完全禁止ではない
- 限定的・文脈補助的に使用

### 実装
- コメント入力欄の下に「感情を添える」オプション
- 5-7種類の基本的な感情（😊😢😮😡🤔👍👎）
- 1コメント1つまで
- オプションなので使わなくても良い

### 禁止事項
- 無制限スタンプ送信
- リアクション乱発
- エンゲージメント稼ぎ目的の絵文字
